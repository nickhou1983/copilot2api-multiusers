02 · API Endpoint

智能路由：按模型能力择优

读取上游 /models 的 supported_endpoints，决定真正请求哪个路径。

  • Claude 优先级 ：/v1/messages → /responses → /chat/completions
  • OpenAI Chat ：模型不支持 Chat 时自动转 Responses
  • OpenAI Responses ：模型不支持 Responses 时回退 Chat
  • Gemini ：先转 OpenAI Chat，再按能力路由；countTokens 本地估算不走上游
03 · 模型 Cache

两层缓存：能力缓存 + Prompt Cache

模型能力缓存

每账号独立缓存 /models，TTL 5 分钟，用于快速判断支持的 endpoint。

Claude Prompt Cache

代理只转发 cache_control 标记并映射 usage，不落地任何 prompt 内容。

Claude 路径cache_control缓存用量映射兼容性
native /v1/messages保留 type/ttl，删 scope保留 cache_read / cache_creation最好
via /responses转换时不保留cached_tokens → cache_read中等
via /chat/completions转换时不保留cached_tokens → cache_read较弱
只有走原生 /v1/messages 时 Prompt Cache 标记才最可能完整传给上游；转 Responses / Chat 后标记丢失，仅保留 usage 命中统计。

04 · 工具调用

三条路径的工具语义保真度

Native Messages 最完整

原生透传，保留 Anthropic tool 语义与流式事件最完整。

Responses 良好

转 function tool，保留 function_call_output 与错误状态。

Chat Completions 基础

转 OpenAI tools / tool_calls，丢失部分语义。

结论：需要最强工具与缓存能力时，让客户端走原生 Anthropic /v1/messages 入口。

06 · 能力差异

Copilot API 的 Claude vs 原生 Anthropic

Copilot 只「转售」对话推理核心；Anthropic 原生还附带服务端工具、文档/缓存治理与细粒度计费元数据。差异来自上游裁剪 + 非原生路径的结构体转换丢字段。

  • 对话 / 自定义工具 ：两者都支持（function 由客户端执行）
  • 服务端内置工具 ：Copilot 不支持（web_search 实测 400）
  • 缓存 / 文档治理 ：仅原生 /v1/messages 路径有机会透传

06 · 能力差异

三点关键结论

① 上游能力远超「转售」

直连实测:文本/流式、function（含并行）、vision base64、PDF 原生 document 块、thinking、cache_control、context_management、citations（带 page_location）、count_tokens、1M 全部上游支持。仅 Computer Use 与 web_search 被拒（400）,URL 图像不支持。

② 路径决定保真度

只有命中原生 /v1/messages（原始字节透传）才可能传递未建模字段；回退到 Responses / Chat 后，结构体重序列化会静默丢字段。

③ 代理已补齐缺口

PDF document 块与 citations 可通过 /v1/messages 透传；context_management 与 count_tokens 已在代理侧补齐(保留字段 + 自动加 beta 头 / 注册 count_tokens 路由)，经代理与直连结果一致。仅剩 cache_control.scope 因上游不支持被代理静默剥离。

建议：需要 PDF document 块、citations、server 工具等高级能力时务必走 /v1/messages 路由（转换路径会因 AnthropicTool 未建模 type、document 块未建模而降级丢弃）；context_management 与 count_tokens 现已透传/路由,经代理与直连等价；Computer Use 与 web_search 在 Copilot 上游确不可用。
07 · Extended Thinking

三路径都支持，但保真度不同

只有原生 /v1/messages 是真正的 Anthropic thinking 透传；另两条路径把 thinking{budget_tokens} 降级映射成各自的 reasoning 机制，再逆向重建回 thinking 块。

路径请求侧：thinking 如何下发
native /v1/messages原始字节透传，thinking{type,budget_tokens} 原样发给 Copilot 最完整
via /responsesbudget_tokens → reasoning.effort 分桶：≥16000=high，≥8000=medium，否则 low；强制 temperature=1、summary=detailed 降级
via /chat/completionsbudget_tokens → OpenAI thinking_budget 扩展字段；effort 不映射 最简
07 · Extended Thinking

响应还原、流式与注意点

响应侧还原成 thinking 块

native：透传。
/responses：reasoning 项 → thinking；encrypted_content+id 编码进 signature（enc@id），多轮可拆回续接。
/chat：reasoning_text → 文本，reasoning_opaque → signature。

流式增量

native：SSE 原样透传。
/responses：reasoning_summary_text.delta → thinking_delta。
/chat：reasoning_text 增量 → thinking_delta，块尾 signature_delta。

注意：非原生路径把精确 budget_tokens 压成 low/medium/high 三档；/responses 还会覆盖你的 temperature 与小 max_tokens（抬到 ≥12800）；signature 被复用为加密推理载体，语义与原生不同。

10 · 上游实测修正

三点上游行为修正

① 1M 上下文已是基础能力

实测 /models:claude-sonnet-4.6、opus-4.6/4.7/4.8 基础模型 max_context_window_tokens 直接为 1,000,000,不存在 -1m 变体 ID。

→ 不再需要 anthropic-beta: context-1m 触发独立模型;代理盲拼 -1m 后缀反而会构造出上游不存在的 ID,导致能力探测落空。已修正为能力感知:基础模型已 1M 则不拼后缀。

② count_tokens 上游 + 代理均已支持

Anthropic 原生有 POST /v1/messages/count_tokens 预计 token 数;Copilot 上游现已提供该接口。

→ 直连上游实测 POST /v1/messages/count_tokens 返回 HTTP 200 {"input_tokens":14},与原生 Anthropic 行为一致；代理已注册该路由(main.go + handleCountTokens),复测同样返回 HTTP 200 input_tokens=14,与直连一致。

③ context_management 上游 + 代理均已支持

实测上游原生支持 Anthropic 上下文编辑:带 anthropic-beta: context-management-2025-06-27 + clear_tool_uses_20250919 触发器,服务端真的裁剪了 tool_use(applied_edits:[{cleared_tool_uses:1}],HTTP 200)。不带 beta 头则 context_management 回显为 null。

→ 代理已修正:normalizeNativeMessagesBody() 保留 context_management 字段,并在检测到该字段时自动注入 anthropic-beta: context-management-2025-06-27 上游头 → 经代理复测同样返回 applied_edits:[{cleared_tool_uses:1, cleared_input_tokens:1211}],与直连一致。

结论:1M 现已下沉为基础能力(代理已去除多余的 -1m 逻辑),count_tokens 与 context_management 上游与代理均已支持且经代理与直连等价。19 项能力对比中代理仅剩 1 处刻意差异:cache_control.scope 上游不支持(400),代理会静默剥离该字段使请求通过。
小结

一句话总结

Copilot2API = GitHub Copilot 的「多协议适配 + 智能路由 + 多账户 + 用量统计」本地网关；缓存与工具能力在原生 Anthropic 路径下最完整，服务端内置工具（含 Web Search）暂不受 Copilot 上游支持。
